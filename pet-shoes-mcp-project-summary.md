# Amazon 关键词分析与可视化项目（sorftime_mcp）

## 项目定位

基于 `sorftime_mcp` 对 Amazon US 关键词 **pet shoes** 做数据化选品分析，并输出业务可读的可视化 HTML 报告。  
目标是把“关键词判断”从主观经验升级为可复用的数据决策流程。

---

## 一、构思与目标

### 1. 业务问题
- 关键词是否有真实需求？
- 竞争是否过度拥挤？
- 市场是否被头部品牌垄断？
- 该方向适合什么体量的卖家进入？

### 2. 分析目标（对应输出）
1. **需求与供给**：月搜索量、购买量、供需比  
2. **竞争结构**：竞争商品数、品牌集中度、头部垄断风险  
3. **进入建议**：适配卖家体量（小/中/大）

---

## 二、通过 MCP 达成构思的方法

### 1. 数据拉取（sorftime_mcp）
- `keyword_detail(amzSite="US", keyword="pet shoes")`  
  获取：月搜索量、周搜索量、竞品数量、首页结构、Top产品/品牌月销占比等。
- `keyword_search_results(amzSite="US", keyword="pet shoes", page=1..n)`  
  获取：自然位 Top 商品、品牌、卖家、单品月销占比，辅助判断头部集中和词意纯度。
- （可选）`keyword_list` 按排名区间补充购买相关字段，校验关键词购买数据可用性。

### 2. 指标计算逻辑
- **供需比** = 月搜索量 / 竞争商品数  
- **品牌集中度（CR5）** = Top5 品牌月销占比之和  
- **头部垄断风险**（规则化）  
  - 高：Top5 品牌占比高 + Top1 单品占比高  
  - 中：Top5 中等、Top1不极端  
  - 低：头部分散
- **卖家体量适配**  
  综合供需比、竞争密度、词意纯度（是否混入非目标产品）给出建议。

### 3. 决策输出框架
- 结论层：值不值得做、风险在哪  
- 策略层：谁更适合做（卖家体量）  
- 动作层：若进入，优先长尾词/投放方向/差异化点

---

## 三、可视化 HTML 输出设计

### 1. 页面结构
- 顶部：关键词、站点、数据时间、核心结论
- 指标卡：月搜索量、竞品数、供需比、CR5
- 图表区：
  - 需求 vs 供给（双轴柱图）
  - 品牌集中度（环形图）
  - 头部商品占比（柱图）
  - 卖家体量适配（卡片/矩阵）
- 底部：口径说明与数据缺失说明（如购买量缺失）

### 2. 图表实现建议
- 技术：`HTML + CSS + Chart.js`
- 原则：
  - 指标卡用于快速阅读
  - 图表只呈现关键业务关系
  - 文案紧贴“可决策结论”

### 3. 产出文件示例
- `pet-shoes-analysis.html`（关键词分析可视化）
- `asin-B01LYITJ4S-negative-feedback-visual.html`（评论洞察可视化）

---

## 四、项目亮点

- 搭建了基于 `sorftime_mcp` 的 Amazon 关键词分析流程，实现从数据拉取、指标计算到结论输出的标准化闭环。  
- 围绕 `pet shoes` 关键词完成供需、竞争、集中度分析，形成卖家体量匹配策略，支持选品与投放前判断。  
- 使用 `HTML + Chart.js` 将分析结果产品化为可视化报告，提升跨角色沟通效率（运营/选品/老板）。  
- 将“经验判断”转为“指标驱动判断”，可复用到更多关键词与类目。
